# 产品方案：求职表达引擎

> 版本：v3.0（2026-07）· 状态：定稿
> 标语：**你比你的简历更好。**（副：先懂你，再写你）
> 本文档是设计主线；ROADMAP.md 为工程记录。

---

## 0. 定论

- **产品只有两件事**：① 挖掘用户背景，帮用户把简历写出来；② 拿着 JD 和简历盘问用户，帮他在真面试前站稳。两件事共享同一份资产（素材库），构成一个闭环：素材喂盘问，盘问暴露素材缺口。
- **停止**："AI 改简历"作为方向已冻结。编辑器/模板/改写/导出降级为免费基础设施（获客入口 + 渲染管线），维护态，不再新增功能。
- **本体**：素材库 —— 挖出来的职业价值资产。简历和定制版是它的投影。
- **边界**：只做表达（挖 → 存 → 讲），到面试为止。**不做分发**：职位匹配、auto-apply、内推、长期职业规划，写死不做。
- **信任原则**：AI 永远放在不需要被信任的位置上 —— **AI 问，不写；AI 照镜子，不评判**。所有事实出自用户之口。
- **交互原则**：不是聊天，是快问快答；**问以文，答以声**（语音输入是 P1 核心，不是增长功能）。
- **入口强假设（一岗一历）**：定制成本趋零后，"每个岗位一份独有简历"成为新常态。主入口 = 贴一个 JD → 访谈 → 为这个岗位而生的简历。成本结构：**挖掘一次性，定制边际近零** —— 首个 JD 深访谈建库，后续 JD 只做投影 + 缺口微访谈（1-3 问）。副入口"聊聊你的经历"兜底无 JD 用户。**自动搜索招聘平台：否决**（撞"不做分发"死线 + 平台封闭无 API + 反爬法律风险）；降摩擦替代 = 浏览器插件/分享菜单（P5）。
- **止损闸门**：P1 上线后盯首访谈完成率与 7 日回访率；用户不愿被访谈即承认假设失败，退回诊断器做小而美或停，不硬撑。

## 1. 为什么：信任结构（本产品最深的地基）

### 1.1 AI 正在杀死简历的信号价值

简历曾经有用，因为它是**有成本的信号**：写得好，部分证明你思路清晰、足够重视。AI 把"写得好"的成本降到零后，流畅的文笔不再传递任何信息 —— HR 面对的是千篇一律的"结果导向、数据驱动、赋能协同"。**"AI 改简历"不只是 commodity，它在参与摧毁简历本身。**

简历里还剩下三样可信的东西：**可验证的事实、具体到无法批量生产的细节、面试盘问下的前后一致。**

> **简历的价值正在从"写得好"迁移到"经得起盘问"。**

### 1.2 用户凭什么信 AI

用户对 AI 的恐惧拆开是三个：它会不会替我撒谎（面试当场社死）；改完还是我吗；后果谁担（AI 不用去面试，我要）。由此得出唯一的信任判据：

> **AI 的输出可信，当且仅当用户能在面试里亲口捍卫其中每一句话。**

两种模式对照：**AI 替你说**（改写/生成/代投）—— 内容源自模型想象，判据不通过；**AI 帮你把你说过的话说清楚**（访谈挖掘）—— 每个事实出自你的嘴，AI 只问和整理，判据通过。这与真人教练的分工完全相同，没有人指责教练摧毁求职者的意义。

### 1.3 人的位置（不可让渡）

用户是**事实的唯一来源**、**取舍的决定者**、**最终考场上独自站着的人**。AI 是提问者和书记员，仅此而已，也足够值钱。

推论三条：
- **出处可溯源（provenance）是一等公民**：简历每条要点可追溯到"你在访谈里说的原话"；
- **营销立场反转**：不说"AI 帮你写简历"，说"AI 写的简历千篇一律 —— 我们帮你写出只有你写得出的那份"。做反 AI 腔的 AI；
- **产品终极承诺**：不是"简历好看"，是"**简历上没有一句你捍卫不了的话**"。

### 1.4 Auto-apply 为什么是死路（顺带回答 Jobright）

投递的价值建立在"投递代表诚意"上。AI 让投递零成本 → 所有人海投 → 招聘方加过滤器/提前笔试/只看内推 → 海投回报归零。公地悲剧，结构性自毁。Jobright 的用户不是信任它，是绝望于别的选择。我们拒绝 auto-apply 是信任层面的结论，不只是能力圈划界。

## 2. 定位

### 2.1 一句话

> **求职表达引擎：帮你把职业价值挖出来、存下来，在两个考场上讲清楚 —— 纸上（简历）和嘴上（面试）。**

结构是特洛伊木马：**入口是简历**（具体、紧迫、可搜索，负责获客）；**本体是陪跑**（持续、可付费的关系，负责留存）。营销永远叫简历工具，产品从第一句对话就是教练。

### 2.2 动机峰值：两个夜晚

产品对准的不是"日常管理职业资产"（自嗨场景），而是两个动机峰值时刻：**投递前夜**（明天要投这个岗）和**面试前夜**（后天面试，慌）。所有入口、文案、首次体验围绕这两个夜晚设计。

### 2.3 目标用户

| 人群 | 真实状态 | 我们的切入 |
| --- | --- | --- |
| **P0 · 3-8 年跳槽者** | 亮点存在但被埋没（挖掘价值最大）；时间稀缺，前 90 秒必须证明值得 | 快问快答 + JD 定向挖掘，付费主力 |
| **P1 · 秋招大学生** | 经历单薄但未被结构化（课程项目/比赛/社团没人帮他讲成 STAR）；焦虑+时间充裕=最有耐心的群体，本来就在手抄 STAR 模板自问自答 | 从零访谈生成，我们把他已经在做的事变成产品 |
| **P2 · 转行者** | 旧经历需要翻译成新行业语言 | 挖掘+可迁移能力重述，客单价最高但量小 |

### 2.4 agent 的能力边界（六个动词，两条禁令）

**做**：问（挖掘）、懂（行情外部视角）、写（把你的话整理成你的简历）、判（对简历的诊断，不是对人的评判）、陪（记得你进行到哪）、练（盘问）。
**不做**：不替用户做人生决定；不编造任何事实（数字只能用户亲口提供）。

## 3. 对象模型：素材库是本体，简历是投影

```
              素材库 Vault（关于「你」的结构化事实）
              ├─ 故事 Story：STAR 四段 + 数字 + 能力标签 + 成色(strong/weak/stub)
              │            + 已问清单 askedFollowups（绝不重问）
              ├─ 量化成果：用户亲口给出的每一个数字（verifiedBy: user）
              └─ 基础档案：教育/任职时间线/技能/联系方式
                              │ 投影 project(vault, jd?)
        ┌─────────────────────┼─────────────────────┐
     通用简历             定制版 A（JD₁）          定制版 B（JD₂）
```

- **迁移**：老用户旧简历自动反向解析为 stub 故事入库，首屏即见诊断（"9 个故事 7 个缺数字"），挖掘价值即刻可见；
- **存储决策（P1）**：vault 是持久化状态的顶层字段（与 resumes 平级，用户级跨简历），schema v2→v3 —— 白坐四趟已建成的车：localStorage 防抖+关页落盘、migrate 体系（v3 迁移顺手做旧简历反解）、登录后整状态云同步（跨设备接力零新代码）、游客先攒素材注册时上云（注册动机的落地）。故事内含 sources[{q, quote, at}] 出处摘录 —— 溯源跟着事实走，不跟聊天走。容量：百条故事 ≈150KB，远低于 2MB 上限；免费层 10 条上限兜住膨胀。
- **素材库即记忆**：对话即焚，事实入库。四层记忆 —— 事实（vault 分层投影：档案全量/故事索引/当前故事展开）、访谈状态（已问/拒答清单）、会话（滚动摘要，P1.5）、偏好（P1.5）。上下文预算 ~4-6K tokens，与素材库规模解耦。不用向量库：几十条故事量级，规则检索精确且零基础设施；
- **素材库即护城河**：聊天记录会滚走，资产会增值；跨会话跨设备随账号走 —— "付费资产"有了物理实体。

## 4. 交互形态

### 4.1 不是聊天，是快问快答

没人想跟 AI 闲聊。形态是**结构化快问快答**：一次一个具体到某故事某缺口的问题（"你写了'负责用户增长'——最拿得出手的一个数字是多少？"），用户答一句（10 秒），画布当场长出一条。耐心的敌人不是时长，是看不到回报 —— **每一步都有可见产出**是意愿假设成立的前提。

### 4.2 问以文，答以声（语音是 P1 核心）

访谈本来就是口头的，用打字做访谈是自相矛盾：

- **说与写的努力差 5-10 倍**（手机打字 ~35 字/分 vs 说话 200+ 字/分）；
- **啰嗦是挖掘的金矿**：口语回答会跑题、带出细节（"哦对了当时还有个事"）—— 挖掘要的恰恰是溜出来的边角料；文字输入在源头掐死原料；
- **不对称双工**：AI 问题用文字（读比听快，3 秒扫完，不做 TTS）；用户回答用语音。两边各取最快通道。

实现纪律：Web Speech API（浏览器原生、零后端、~百行，过复杂度铁律）；**转写先落输入框、用户过目可改再发送**（纠 ASR 错 + 守"每个字出自用户"铁律 —— 这个设计同时降低了精度要求：ASR 只需"好到值得改"）；打字永远保留兜底；隐私诚实标注（浏览器 ASR 上传音频给浏览器厂商）。

**升级路径带触发器**：P1 本地记录**转写修改率**（发送前编辑距离）。修改率高或 iOS 占比大 → P1.5 换云端 ASR（MediaRecorder → /api/asr → 讯飞/腾讯/Whisper，走现有计量后端，音频分钟并入付费额度），前端只换引擎。选厂标准：中文口语准确率 > 流式 > 单价 > 部署地合规。端侧模型（whisper WASM）否决：中文差、40MB+、手机慢。

### 4.3 访谈引擎（缺口驱动的选题循环）

```
算缺口(coverage × JD差距 × 量化密度) → 选题(最值钱的一个) → 问一个具体问题
→ 用户口答 → 结构化入库(upsert_story) → 素材达标即落纸(画布高亮) → 回到算缺口
```

纪律：一次一问，禁止"还有什么要补充的吗"式空问；回答即资产（跑题也入库为 stub）；随时可被编辑指令打断（访谈是默认态不是强制态）；连续 3 问一次"喘息"（亮出刚挖到的东西）；15 分钟自然收束并给出**挖掘成果单**（"你说了 23 句话，我挖出了 4 个你从没写进简历的数字"——这张单子是口碑截图时刻）。

**两种访谈深度**：首次（素材库冷启动）= 完整深访谈；后续每个 JD = **投影 + 缺口微访谈**（素材库覆盖不到的才问，1-3 问）。由此形成用户可感知的复利飞轮：处理的 JD 越多 → 素材库越厚 → 下一份定制版越快（"第 5 份定制版只花了 90 秒"）—— 素材库护城河的可见化。

### 4.4 简历盘问（原"模拟面试"，verdict-free 重设计）

用户不信 AI 的**评判**（凭什么打分？没面过人、没有 skin in the game）——但**练习的价值不依赖对陪练的信任**：问题好不好是自证的（答不上来自己当场知道），卡壳是第一人称证据。设计三条：

- **AI 永远不评判**：不打分、不说"表现如何"、不预测通过率；
- **AI 只做两件事**：用你的素材出追问（每一问指向简历上的一句话）+ 当镜子（陈述客观事实："你刚才的回答没有提到数字"）；
- **判断权归用户**：答不上来的那一刻，你自己就是评委。

价值主张收窄为：**在真面试之前，找出简历上你捍卫不了的句子。** 它是 §1.3 终极承诺的质检工序，也是素材闭环的回路（盘问暴露的缺口回流访谈）。AI 相对真人陪练的优势恰恰是无社交成本：可以卡壳一百次、凌晨两点练、没人看见你丢脸 —— 对照组不是真人教练，是"从不开口练习，第一次讲自己的故事就是在真面试上"。

### 4.5 界面：双面结构（舞台 + 工作台）

语言是核心 ≠ 页面全是对话。产品有两种时刻，各配一面：

**访谈舞台（新建，语音的家）**：访谈/盘问进行时的全屏专注态 —— 问题大字号居中（一屏一问，不是气泡流）、麦克风是最大交互元素（转写可改再发）、挖掘回执滑入、右下角简历缩略图实时长出新行（努力悖论的对冲落在像素上）。**一台两用**：挖掘访谈与简历盘问是同一组件的两种模式。天然即移动端形态。

**工作台（现有，保留不动）**：画布+精修+导出，审阅微调时用。已是免费基础设施，零重构。现有对话栏降级为记录/编辑指令入口，不再承载访谈。

**入口倒转**：首次进入 → 引导三问 → 直接进舞台第一问（15 分钟 aha 发生在舞台上）；投递前夜 → 贴 JD → 舞台定向访谈；面试前夜 → 舞台盘问；舞台退出 → 工作台看成品导出。用户落地在"一场对话，顺便产出了文档"，而不是"一个带聊天的编辑器"。

| 区域 | 定位 |
| --- | --- |
| 访谈舞台（新） | 会话时刻的主场：问题卡 + 语音应答 + 回执 + 成果单 + 简历缩略图 |
| 状态条 | 挖掘仪表：故事 n（成色分布）· 数字 n · 对 JD 缺口 n → 点开素材库面板 |
| 素材库面板（新） | 故事列表 + 成色 + STAR 详情 + "继续挖这条"；付费资产的可视化容器 |
| 画布/精修/导出 | 工作台，保留不动（手动兜底 + 渲染管线） |

### 4.6 平台策略：跨设备接力（手机挖，电脑出）

"语音天然属于手机"与"没人在手机上写简历"不冲突 —— 按活动分设备：**挖**（访谈/盘问，说话+私密场景=两个夜晚在家）归手机；**讲·纸**（审阅/导出/投递校招门户）归电脑；素材库云同步就是接力线。产品时刻：**"睡前聊一聊，醒来简历就绪。"**

- **接力零新架构**：素材库随账号同步（已建成）；桌面端加"用手机继续访谈"二维码直达舞台；
- **注册的天然理由**："想在电脑上导出昨晚聊出的简历 → 登录" —— 具体、当下、自己撞上来的转化动机，强于抽象的"云备份"；
- **分发闭环**：小红书/抖音点击落在手机 → 落地即舞台即 aha → "到电脑导出"完成注册接力；
- **克制**：P1 不做原生 App，响应式 Web + 舞台移动优先即可（Web Speech API 移动端 Chrome 可用）；桌面舞台同样可用（笔记本麦克风），不强迫换设备；原生化 P5 后拿数据再议。

## 5. 竞争

### 5.1 市场事实（2026-07 调研）

- **简历赛道四代**：Gen1 写作器（Kickresume/超级简历：按职位生成+语料优化）→ Gen2 诊断器（Rezi/Jobscan：ATS 评分）→ Gen3 套件（Teal/ResumeUp 6合1）→ Gen4 求职 copilot（Jobright auto-apply，信任薄、结构自毁）。
- **两件事都有人做，但都是错的姿势**：挖掘式写简历只有浅版（ResumeUp 问三五个问题的 onboarding 向导）；AI 模拟面试国内已卷（牛客/牛面/鹅来面）但全是**题库出题+评分报告** —— 恰好是"AI 评判"这个不被信任的模式，且与用户简历零关联。**深访谈+素材资产没人做；两件事的闭环没人做。**
- **文档赛道三段演化**（Notion/飞书/Office）：嵌入助手 → 知识层 → agent 化，AI 从功能变本体 —— 方向性佐证。
- **人肉版价格表**（需求侧最硬证据）：简历优化单次 **¥458**（40 分钟电话沟通+调整交付+一月内再改 —— 逐字就是我们 P1 的人肉版）；求职陪跑 ¥3999/月；机构套餐 ¥2.3-5.7 万。市场大到有"割韭菜"报道 = 需求真实、供给混乱、可被标准化产品替代。

### 5.2 诚实的现状与后发优势

**今天我们没有优势** —— 分发为零、资产未建、流程未上线，四道墙三道是图纸。这是 day-0 常态不是死刑。后发优势三条，全部依赖"深挖掘是真痛点"才兑现：

1. 市场教育费别人付了（"AI 简历""AI 面试"认知已建立）；
2. **在位者被自己的资产锁死**：牛客的题库+八股+B 端初筛业务决定了它必须"题库出题+打分"（否则否定自家产品）；超级简历锁在优化+导师现金流上，做不了"改写免费、挖掘收费"。后发者最大的优势是没有需要背叛的资产；
3. 时机：访谈级对话 2024 后才可行，在位者全是前 LLM 内核 + AI 贴皮。

### 5.3 竞争纪律

**不进它们的考场**：模板数量、ATS 评分、语料优化 —— 一场都不打。对手不是超级简历，是"求职时身边没有懂行的人"这个空缺和 ¥500/小时的真人教练。**做新品类的第一名，不做旧品类的第十名。**

### 5.4 平台吞噬（为什么不被 skill/MCP 取代）

我们的话术一个 skill 就能抄走 —— 所以话术不算资产。四道抄不走的墙：**分发**（会装 MCP 的人 <1% 求职人群）、**专用界面**（对话+画布+diff+撤销+导出的工作台，聊天窗口物理上装不下）、**结构化资产**（素材库 schema + 培育 UI，通用助手的记忆不为垂直场景负责）、**流程责任**（焦虑的非专家要的是被带着走 + 有人对结果负责）。
**路线图过滤器**：每个新功能先问"skill/MCP 能不能抄走"，能抄的只做必要投入。
**对冲棋（P5）**：自己做成 MCP server —— 入口开在别人家客厅，资产锁在我们家保险柜。

## 6. 三个假设与验证（反自嗨机制）

| 假设 | 内容 | 状态 | 验证方式 |
| --- | --- | --- | --- |
| **能力假设** | AI 快问快答挖得出真亮点吗 | **已初步确认**（Wizard-of-Oz 口头测试；标准=十问内挖出 ≥2 个用户没想过写进简历的数字/细节）；AI 不需要像真人，只需要打败空白页 | 持续：每轮真实访谈的成果单数据 |
| **意愿假设** | 用户肯不肯答（最大自嗨风险=努力悖论：付 ¥458 的人愿意被访谈，免费用户未必） | **未验证，P1 的全部目的** | P1-min 上线 → 20-50 个真实求职者 → 首访谈完成率 + 7 日回访率 |
| **需求假设** | "没人做"是空位还是坟场 | 半验证：¥458 人肉版存在（需求为真）；但其中"真人确定感"占比未知 —— **全案最脆弱一环** | 意愿假设通过后看付费转化 |

**努力悖论的三个对冲**：两个夜晚定位（动机峰值）、语音输入（努力降 5-10 倍）、即时落纸（每答一句画布长一条）。

**止损**：意愿数据不行 → 按闸门退回诊断器做小而美或停。自嗨的定义不是想错了，是不肯让现实纠正 —— 停止条件白纸黑字在此。

## 7. 付费设计

**计费的是教练，不是笔。** 编辑器/模板/导出永久免费（commodity，负责获客与口碑）。

| 层级 | 内容 | 定价锚 |
| --- | --- | --- |
| 免费 | 全部编辑器 · 每日 N 轮轻访谈（够尝到"被挖出一个数字"）· 素材库 10 条上限 | 获客+价值时刻 |
| Pro 订阅 | 无限访谈 · 素材库无上限 · JD 定制投影 · 简历盘问 · 云同步 | 人肉版 ¥458/次、陪跑 ¥3999/月 → 我们 ¥29-49/月，价差 10 倍+ |
| 冲刺包 | 针对一个 JD 的完整挖掘+定制+盘问，7 天 | ¥19-29/次，接住不订阅的临时求职者 |

转化时机（价值先行）：首个数字被挖出 → 回执卡转化；素材库到上限 → 资产型转化；JD 投影 → 结果型转化。
**北极星指标：每周被挖掘出的 strong 故事数。**

## 8. 复杂度铁律（优先级高于本文档一切功能描述）

1. **模型做判断，代码只记账**：永远一个 agent 循环；不引入嵌套 agent/planner/编排框架/状态机库
2. **状态就是普通 JSON**：vault 能 console.log，走同一套 localStorage+云同步
3. **上下文 = 纯函数**：`buildContext(vault, chat) → string`，可单测可打印
4. **每期一个新概念**：P1 只有 vault；其余问"现在真的疼了吗"再上
5. **画图测试**：需要画图才能解释的设计，砍
6. **测试网即安全网**：新概念必须带全链路 e2e —— 维护者不需要在脑中 hold 住系统

## 10. 风险

| 风险 | 缓解 |
| --- | --- |
| 努力悖论（最大）：用户没耐心答 | 语音 + 两个夜晚 + 即时落纸 + 3 问一喘息 + 15 分钟收束；P1 唯一使命就是测它 |
| ¥458 里"真人确定感"占比未知 | 承认拿不到这部分；靠成果单的即时证据替代权威背书 |
| 模型提问质量不稳 | 选题走确定性规则，模型只负责表述+结构化；必要时提问环节换强模型（低频可控） |
| 内容真实性 | 数字只能用户亲口给；出处可溯源；转写过目再发 |
| ASR 识别质量/隐私 | 转写可改再发兜底；隐私诚实标注；打字永远可用 |
| 老用户断裂 | 旧简历反解入库；编辑指令全保留；访谈是默认态非强制态 |
