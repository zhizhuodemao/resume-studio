# 产品方案：从「改写工具」到「挖掘教练」

> 版本：v1.0 草案（2026-07）· 状态：待评审
> 本文档回答四个问题：为什么需要 agent、产品到底做什么、凭什么赢、用户为什么付费。
> 评审通过后，本文档取代 ROADMAP.md 成为设计主线；ROADMAP.md 降级为工程记录。

---

## 1. 问题诊断：现在的产品为什么立不住

当前形态是「简历编辑器 + 会改写的 agent」。它的根本缺陷：

| 缺陷 | 说明 |
| --- | --- |
| **改写是 commodity** | 把简历粘进 ChatGPT 说"润色一下"，效果与我们相差无几，且免费。改写能力构不成任何壁垒 |
| **改写不产生增量信息** | 平庸的输入，润色后是措辞漂亮的平庸。用户简历差的根因不是措辞，是**素材缺失**：值钱的经历没写、数字没算过、亮点埋在角落 |
| **编辑指令是低频需求** | "换个模板""改得更量化"是一次性动作，做完即走。没有留存钩子，没有付费理由 |
| **agent 没有回答"为什么非我不可"** | 现在 agent 的全部能力（改内容/换模板/调排版）用户用免费工具+手都能完成，只是慢一点 |

**结论：打字员没人雇，面试官才值钱。**

## 2. 产品定位

### 2.1 一句话定位

> **一位随时在线的职业访谈教练：通过对话把你做过的值钱的事挖出来、讲清楚 —— 简历只是访谈的副产品。**

### 2.2 我们相信的三个判断

1. **绝大多数人不知道自己的经历里什么值钱。** 他们做过有分量的工作，但一笔带过；有可算的数字，但从没算过；最亮的成果埋在第三行。这是简历差的第一根因，措辞是第二位的。
2. **自我挖掘无法独自完成。** "你不知道自己不知道什么"——需要一个会提问的外部视角。市场已验证：真人简历教练时薪 ¥500–2000，卖的就是一小时的好问题。
3. **访谈必须是 agent。** 多轮、有状态、每一步要判断"下一个问题问什么最值钱"、要听出回答里藏着的成果并追问数字 —— 这是唯一非 agent 不可的环节。改写恰恰不需要 agent。

### 2.3 目标用户（按优先级）

| 人群 | 痛点 | 我们的切入 |
| --- | --- | --- |
| **P0 · 3-8 年经验的跳槽者** | 做过很多事但简历写不出差异化；对着 JD 不知道怎么改 | 挖掘访谈 + JD 定制投影，付费主力 |
| **P1 · 应届生/实习生** | 没东西可写（其实有——课程项目/比赛/社团都没被挖掘） | 从零访谈生成，价格敏感，走免费+轻付费 |
| **P2 · 转行者** | 旧经历如何翻译成新行业的语言 | 挖掘 + 可迁移能力重述，客单价最高但量小 |

## 3. 对象模型：素材库是本体，简历是投影

这是本次重构最核心的一变。

```
                    ┌──────────────────────────────┐
                    │       素材库 Vault（本体）      │
                    │  关于「你这个人」的结构化事实     │
                    ├──────────────────────────────┤
                    │ · 故事 Story（STAR 结构）       │
                    │   情境/任务/动作/结果 + 数字     │
                    │   标签：能力项、适配岗位方向      │
                    │ · 量化成果 Metric               │
                    │   挖出来的每一个数字及其出处      │
                    │ · 基础档案 Profile              │
                    │   教育/任职时间线/技能/联系方式   │
                    └──────────────┬───────────────┘
                                   │ 投影 project(vault, jd?)
              ┌────────────────────┼────────────────────┐
              ▼                    ▼                    ▼
        通用简历（无 JD）      定制版 A（JD₁）        定制版 B（JD₂）
        选故事·排优先级       按 JD 选材+改叙述角度   同左
```

### 3.1 数据结构草案

```js
vault: {
  profile: { /* 现 basics + education + skills 的事实层 */ },
  stories: [{
    id, title,                 // "重构 CRM 线索流程"
    situation, task, action, result,   // STAR 四段
    metrics: [metricId],       // 关联的量化成果
    skills: ['数据分析', '跨部门协作'],   // 能力标签
    source: 'interview' | 'imported',   // 来源：访谈挖出 / 从旧简历解析
    strength: 'strong' | 'weak' | 'stub',  // 成色：完整/缺数字/只有一句话
    askedFollowups: [...],     // 已追问过的问题（防重复审讯）
  }],
  metrics: [{ id, value: '转化率 +27%', context, verifiedBy: 'user' }],
}
resumes: [...]                 // 现有多文档结构保留，但内容字段
                               // 可标注 sourceStoryIds —— 从哪些素材投影而来
```

### 3.2 迁移路径

- 现有简历数据不丢：首次进入新版本时，**旧简历自动反向解析入库**（每条工作经历要点 → 一个 `stub` 成色的故事），访谈从"把 stub 养成 strong"开始 —— 老用户第一眼就能看到"你有 9 个故事，其中 7 个缺数字"，挖掘价值即刻可见。

## 4. 核心交互流程

### 4.1 访谈引擎（产品的心脏）

访谈不是自由聊天，是**缺口模型驱动的选题循环**：

```
┌─→ 1. 算缺口：coverage(vault) × JD 差距 × 量化密度
│   2. 选题：挑当前最值钱的一个缺口（规则+模型混合）
│   3. 提问：一次只问一个具体问题（"重构线索流程那件事，
│      上线后转化率具体变了多少？"）
│   4. 听答：用户随口回答（语音/文字，允许口语化、碎片化）
│   5. 结构化：agent 从回答中抽取 → 写入/更新 Story 与 Metric
│      （工具：upsert_story / add_metric，替代现在裸的 update_resume_content）
│   6. 落纸：素材达到 strong 成色时，同步投影到简历对应板块
│      （画布实时可见"你刚说的话变成了简历上的这一条"）
└── 7. 回到 1
```

关键设计约束：

- **一次一个问题**，问题必须具体到某个故事的某个缺口，禁止"还有什么要补充的吗"这类空问
- **回答即资产**：哪怕用户跑题说了段别的经历，也要 upsert 成新 stub，绝不丢弃
- **随时可打断**：用户仍可下达任何编辑指令（现有全部工具保留），指令完成后访谈从缺口模型继续 —— 访谈是默认态，不是强制态
- **防审讯感**：连续 3 问后主动给一次"喘息"（展示刚才挖到了什么），每次会话有自然收束点

### 4.2 记忆架构（访谈引擎的前提）

上下文窗口有限且昂贵，长期访谈**不能依赖对话历史作为记忆**。原则：**对话即焚，事实入库** —— 每轮对话中值钱的信息立刻结构化存储，之后每一轮的上下文都是从结构化状态**重建**出来的，而不是拖着完整聊天记录。

四层记忆，各有归宿：

| 层 | 内容 | 存储 | 进上下文的方式 |
| --- | --- | --- | --- |
| **事实记忆（长期）** | 素材库：故事/数字/档案 | vault（本地 + 云同步，随账号走） | 分层投影：档案全量；故事默认只给索引（标题+成色+标签）；**当前选题的故事展开全文 STAR** |
| **访谈状态（中期）** | 缺口模型现值、每个故事已问过的问题、用户拒答/跳过清单（绝不重问）、会话计数 | vault.interview | 摘要一行注入："已问 23 题，故事 X 已追问过数字被拒" |
| **会话记忆（短期）** | 滚动摘要：每 K 轮把旧轮次压缩为要点（关键决定/未完话题），保留最近 6-10 轮原文 | 随对话持久化 | summary + 最近轮原文（替代现在粗暴的 slice(-14) 截断） |
| **偏好记忆** | 用户表达过的口味："别用夸张的词"、"不想出现上家公司名" | vault.preferences | 全量注入（条数少、违反代价大） |

**上下文预算**：每轮 prompt ≈ 角色规则 + 档案 + 故事索引 + 当前故事全文 + 缺口摘要 + 偏好 + 会话摘要 + 最近轮 ≈ 4-6K tokens，与素材库规模基本**解耦**（素材涨十倍，上下文只涨索引行数）。

**为什么不用向量库/RAG**：单用户素材量级是几十条故事，选题引擎本来就知道当前在挖哪条 —— 规则检索（选中的展开、其余给索引）精确且零基础设施。等素材库出现跨用户知识（比如行业问题库）再考虑检索层。

**记忆不是为了"装得下"，是三件事**：① 成本与长上下文质量衰减（模型对中段信息注意力差）；② **跨会话**——关掉浏览器一周后回来，教练记得一切；③ **跨设备**——记忆在 vault 里随云同步走，这也让"素材库=付费资产"的说法有了物理实体。

### 4.3 三个入口流程

| 场景 | 流程 |
| --- | --- |
| **从零开始**（应届/无简历） | 引导 3 问（方向/阶段/目标 JD 可选）→ 直接进访谈 → 画布从空白开始"长出"简历 —— 首访谈 15 分钟给出可导出的初版，这是产品的 aha moment |
| **带旧简历来** | 粘贴/导入 → 反向解析入库（stub 故事）→ 首屏亮出诊断"9 个故事 7 个缺数字，预计 20 分钟补齐"→ 进访谈 |
| **对着 JD 投递** | JD 入档（现有）→ 差距分析驱动定向访谈（"JD 要求团队管理经验，你的素材库里没有——带过人吗？"）→ 素材补齐后一键投影定制版 |

### 4.4 界面形态调整（在现有壳层上演进，不推倒）

| 区域 | 现在 | 变为 |
| --- | --- | --- |
| 左栏对话 | 等待指令的助手 | **访谈主线**：问题卡、挖掘回执（"✦ 已存入故事：重构线索流程，+1 个数字"）、诊断卡 |
| 状态条 | 体检/教练/岗位 | **挖掘仪表**：故事 n 个（strong/weak/stub 分布）· 量化成果 n 个 · 对 JD 缺口 n 个 —— 点开即素材库面板 |
| 素材库面板（新） | 无 | 故事列表（成色标记、点击查看 STAR 详情、"继续挖这条"按钮）；这是付费资产的可视化容器 |
| 画布 | 编辑对象 | 投影结果 + 实时反馈面（访谈中新素材落纸时高亮，已有机制） |
| 精修面板 | 保留 | 保留不动（手动兜底） |

## 5. 付费设计

### 5.1 原则

**计费的是教练，不是笔。** 改写/模板/排版/导出永久免费（它们是 commodity，正好做获客与口碑）；挖掘访谈、素材库、定制投影是付费品。

### 5.2 分层

| 层级 | 内容 | 定价思路 |
| --- | --- | --- |
| **免费** | 全部编辑器/模板/导出 · 每日 N 轮轻访谈（够尝到"被挖出一个数字"的 aha）· 素材库上限 10 条故事 | 获客 + 体验价值时刻 |
| **Pro（订阅）** | 无限访谈 · 素材库无上限 · JD 定制投影无限 · 多简历云同步 | 锚定真人教练 ¥500/小时：定价 ¥29-49/月即有 10-20 倍价差的说服力 |
| **冲刺包（单次）** | "面试冲刺"：针对一个 JD 的完整挖掘+定制+求职信，7 天有效 | 面向不想订阅的临时求职者，¥19-29/次 |

### 5.3 转化时机（价值先行，沿用现有漏斗哲学）

1. 免费轻访谈挖出第一个数字 → 挖掘回执卡即转化点（"这样的缺口还有 6 个 → 解锁完整访谈"）
2. 素材库到达上限 → 资产型转化（"你的故事值得被完整保存"）
3. JD 定制投影 → 结果型转化（展示匹配度提升预估）

### 5.4 北极星指标

**每周被挖掘出的 strong 故事数**（代理"用户价值被讲清楚的量"）。
过程指标：首访谈完成率、访谈→素材转化率（每 10 问产出几条 strong）、素材库→定制版投影次数、免费→付费转化率。

## 6. 竞争力总结

| 维度 | 别人（含 ChatGPT 裸用） | 我们 |
| --- | --- | --- |
| 卖什么 | 更快地写字 | 把你自己讲清楚 |
| 价值来源 | 重新包装已有信息 | 挖掘增量信息（数字、故事、差异化） |
| 护城河 | 无（提示词即可复制） | 素材库资产（越用越厚，迁移即归零）+ 访谈选题引擎（缺口模型是数据与工程的复利） |
| 价格锚 | 免费工具 | 真人教练 ¥500+/小时 |
| 留存钩子 | 无（改完即走） | 每次投递回来补素材；素材库是职业生涯级资产 |

已有工程恰好是地基：agent 真循环（Wave 1）= 访谈执行器；教练状态机（Wave 4）= 缺口模型雏形；JD 工作台（Wave 3）= 定向访谈的靶子；薄后端计量 = 付费墙的秤。**主从倒转，而非推倒重来。**

## 8. 风险与开放问题

| 风险 | 缓解 |
| --- | --- |
| **访谈疲劳**：用户没耐心被连续提问 | 一次一问 + 3 问一喘息 + 每问都即时可见回报（画布落纸/回执卡）；会话 15 分钟自然收束 |
| **模型提问质量不稳**（deepseek-chat 上限） | 选题走规则缺口模型（确定性），模型只负责把选题表述成好问题 + 结构化回答；必要时提问环节换更强模型（成本可控，低频） |
| **挖掘出的内容真实性**（用户顺着 AI 夸大） | 沿用绝不编造原则：所有数字须用户亲口提供，agent 只追问不建议数值；素材标注 verifiedBy |
| **老用户体验断裂** | 反向解析让老数据即刻进入新体系；编辑指令全保留；访谈是默认态不是强制态 |
| **开放问题** | ① 语音输入优先级（访谈场景强需求，但工程量大）② 素材库是否云端加密（隐私卖点 vs 服务端能力）③ 冲刺包与订阅的左右互搏 |
