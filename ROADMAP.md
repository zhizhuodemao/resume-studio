# Resume Studio 改进路线图

> 本文档描述从当前版本走向成熟商用产品的改进方向与优先级。
> 更新时间：2026-07（P0–P3 已全部落地，详见「现状」；剩余项集中在 P4 商业化）

## 现状（已完成 ✅）

**编辑器（P0）**

- 数据 schema 版本化（v2）+ v1 自动迁移 + 防御性规范化
- 多份简历管理（新建 / 复制 / 重命名 / 删除 / 切换，每份独立模板与排版设置）
- 撤销 / 重做（60 步历史、输入合并、Ctrl+Z / Ctrl+Shift+Z，AI 修改同样可撤销）
- JSON 导入 / 导出（兼容旧版数据格式）
- 自定义板块（证书 / 语言 / 获奖 / 志愿 / 自定义标题，8 套模板原生渲染）
- 轻量富文本（**加粗** / *斜体* / 链接，安全解析非 innerHTML）+ 选区工具条
- 板块与条目拖拽排序（保留箭头按钮作为无障碍回退）

**排版（P1）**

- A4 / US Letter 纸张、页边距三档、字体五档（跟随模板 / 黑 / 宋 / 楷 / 仿宋）
- 一键压缩到一页（有界排版压缩因子）+ 页数指示
- 照片编辑（拖动定位 / 缩放 / 方形裁剪重导出）

**智能（P2，竞争力核心）**

- L1：AI 润色、整份翻译（可撤销）、一句话生成要点、根据简历生成简介 —— 全部带防编造约束与应用前预览
- L2：规则版简历体检（量化率 / 弱动词 / 完整度等，0 AI 成本）+ JD 匹配分析（匹配度 / 缺失关键词 / 建议）
- L3：JD 定制 agent（一键产出岗位定制版新简历副本，原件不动）+ 简历教练 agent（访谈式多轮对话，结构化补丁直接写回简历，可撤销）

**输出与配套（P3）**

- Word（.docx）导出、纯文本导出
- 求职信模块（附加页 + AI 初稿 + 打印分页）
- 预览点击 → 编辑区定位（双向联动）
- 移动端编辑 / 预览标签切换
- PDF 文件名跟随简历名称

**AI-first 重构（M1–M4）**

- 统一 AI 助手（左主栏对话流）：教练访谈、自然语言操作（内容 / 模板 / 颜色 / 排版 / 纸张 / 板块 / 翻译，function calling 接管全部编辑器操作）、JD 粘贴即分析、确认后生成定制版 —— 原教练抽屉 / 命令浮层 / 体检抽屉合并为一条对话
- 常驻体检评分置顶对话流；每轮 AI 修改产生变更芯片 + 单轮撤销（快照恢复）
- 表单编辑器降级为「精修」侧滑面板（工具栏开关 / 点击画布任意板块直达对应卡片）
- 目标驱动入口：引导页可粘贴 JD，进入后助手自动完成首轮匹配分析
- AI 设计语言 v1：统一 ✦ 标记与渐变 token（--ai-gradient）、生成中脉冲、AI 修改后画布光晕
- SSE 流式：助手回复打字机逐字渲染、tool_calls 增量分片拼装后执行、「正在执行操作」中间态；Vercel 代理改 Edge Runtime 流式透传；对 JSON 响应自动回退（兼容测试 mock 与不支持流式的代理）

**AI-first 收敛 + 商业化漏斗（第二轮重构）**

- AI 全聚合：全站唯一 AI 入口 = 助手对话流（编辑器散落的润色/生成按钮改为 ✦ 一键把上下文交给助手；求职信起草也走助手工具 set_cover_letter）
- 工具栏做减法：模板 / 排版 / 色板 / AI 菜单全部撤出常驻 UI —— 外观决策主路径 = 对话 + onboarding 推荐；手动控制收进精修面板「外观」页签
- 权限分层：游客（每日 N 次 AI 试用，按 IP 限量，默认 5）→ 免费用户（token 额度 + 云同步）→ pro（无限额度，plan 字段就绪等支付）
- 转化漏斗（价值先行）：游客先体验 AI 魔法 → 助手顶部实时显示剩余次数 → 首次成功修改后一次性转化卡（云端备份卖点）→ 额度用尽硬门槛"注册继续" → onboarding 提供"已有账号？登录恢复"

**Agent 化四波重构（Wave 1–4）**

- Wave 1 真循环：工具执行结果（ok/changed/no-op+原因）回喂模型，最多 8 轮自我修正直至完成；行动实时流（⟳→✓/✗）；结构化 diff 回执（板块·字段，旧红删除线→新绿，可展开）；画布改动板块高亮 + diff 行点击定位
- Wave 2 对话体验：✦ 头像与行动/回复/错误三形态、审阅模式（自动应用 / 先确认后采纳）、对话按文档持久化、错误可重试、随状态变化的建议芯片
- Wave 3 主动性与深度任务：每轮后持续建议 + 体检一键修复芯片；plan_steps/mark_step_done 复杂任务清单（可视打钩）；画布划选文字 → ✦ 改这段；JD 工作台（目标岗位随文档持久化、匹配度分析、缺失关键词一键融入）
- Wave 4 教练状态机：coachCoverage 规则引擎（五区域 缺失/弱/完成 覆盖地图），面板点击定向访谈，覆盖摘要注入模型上下文使提问始终瞄准短板
- 顶层错误边界：任何运行时错误显示刷新提示而非白屏

**求职表达引擎 P1-min（产品主线切换后的首个交付，见 PRODUCT.md）**

- 素材库 vault（schema v3）：故事（STAR + 用户亲口数字 + 出处原话 + 已问/拒答清单）作为用户级资产与 resumes 平级；v2 状态自动迁移并把旧简历反解为存根故事；成色由内容判定（stub/weak/strong）
- 访谈舞台：全屏专注态（一屏一问大字号、回执滑入、简历缩略图实时落纸、成果单）；入口三条路（贴 JD 定向访谈 / 贴旧简历 AI 解析入库 / 直接聊）+ 中英语言选择（联动界面/ASR/提问语言）；首次进入直达舞台（入口倒转）
- 语音：Web Speech API 零后端；连续模式（问题落下自动开麦、停顿 2.8s 自动发送）/ 手动模式（按键+过目再发）双轨；ASR 修改率传感器（P1.5 换云端 ASR 的触发器）
- 访谈引擎：interview 模式提示词走同一个 agent 循环；确定性选题（缺数字→补齐→新领域；问题级拒答不废整条故事）；upsert_story 工具（只记用户亲口事实）
- 一段对话两种镜头：舞台与侧栏共享按文档持久化的同一份对话；跨关闭/刷新原地续且零额外调用；舞台带回答回显与记录抽屉
- 状态感知：每轮上下文从最新状态现场重建 + 变更哨兵（手动修改在下一轮被显式告知模型）

**工程与基建（横切 + P4 就绪）**

- 70 个单元测试（vitest）+ 45 个端到端测试（Playwright，覆盖全部核心用户旅程；AI 用按调用序状态机 mock，关键行为断言到被捕获的系统提示词级别）
- React 错误边界（模板渲染崩溃不再白屏）
- GitHub Actions CI（单测 + 构建 + e2e）
- 薄后端（server/，Express + Node 内置 SQLite）：邮箱注册登录（scrypt+JWT）、云端简历存档（随账号绑定、自动同步、换设备恢复）、AI 计量代理（登录门控 + 逐次 token 记账 + 可配免费额度 AI_TOKEN_LIMIT）——收费的数据基础已就绪
- 单进程部署：node server/index.js 同时服务静态产物与 API，可部署任意 Node 托管

## 竞争定位（不变）

1. **免费无水印导出**获客（边际成本为零的 local-first 架构）；
2. **隐私即卖点**（简历不上传服务器）；
3. **AI 简历教练**变现（工具卖不上价，服务可以）。AI 是唯一有真实边际成本的能力，也是唯一计费点。

## 明确缓做项（有理由的延期）

| 事项 | 理由 |
| --- | --- |
| AI 内容标记（AI 起草待确认的可视区分） | 需要内容级溯源数据结构；流式已就绪，可随下轮内容层改造做 |
| 目标看板（投递状态管理） | 多简历 + 定制版命名已承载基础形态；看板等真实用户投递数据需求验证 |
| 手动分页符 | 连续页预览无法可视化分页效果，需先做真分页预览；条目级防拆已默认生效 |
| 服务端 PDF 渲染 | 需要后端算力（Puppeteer 服务），归入薄后端阶段；当前打印管线在 Chrome/Edge 上质量已达标 |
| ATS 友好度专项检查 | 关键词层面已被 JD 匹配覆盖；格式层面等海外线（Letter 已就绪）有真实用户再做 |
| 模拟面试官 agent | 延伸产品，教练 agent 验证后再扩展 |
| LinkedIn / PDF 解析导入 | 需要服务端解析能力，归入薄后端阶段 |

## 优先级逻辑（回顾）

1. **是不是地基？** → P0 全部完成，后续功能都建立在 schema v2 / 多简历 / 撤销之上（教练补丁可撤销、定制版落到新文档，都是这层地基的直接回报）
2. **要不要后端？** → 所有纯前端项已做完；剩余项全部依赖后端或外部资源，聚焦为一张 P4 决策清单
3. **是不是竞争力？** → 三个差异化支点中，前两个（免费导出、local-first）是架构自带，第三个（AI 教练）已上线 v1
