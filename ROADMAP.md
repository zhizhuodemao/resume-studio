# Resume Studio 改进路线图

> 本文档描述从当前版本走向成熟商用产品的改进方向与优先级。
> 更新时间：2026-07（随迭代滚动更新）

## 现状

已完成的能力：

- 8 套视觉模板（现代 / 经典 / 双栏·深 / 双栏·浅 / 时间线 / 校招 / 极简 / 创意），排版三参数（字体 / 字号 / 密度）对全部模板生效
- 实时 A4 预览 + 浏览器打印管线导出 PDF（矢量文字，多页分页已验证）
- 示例内容库：5 职业族 × 社招/校招 × 中英双语，载入时联动推荐模板与板块顺序
- 新用户引导、职业族联动的输入占位提示
- 中英双语界面、主题色系统、localStorage 自动保存
- **AI 助手原型（DeepSeek）**：AI 润色（逐条改写，防编造 + 【补充数据】占位 + 应用前预览）、AI 整份翻译（结构保护 + 可撤销）；key 经 Vite 开发代理注入，浏览器不可见
- 代码已托管 GitHub（`zhizhuodemao/resume-studio`），Apache 2.0

## 竞争定位（先于一切功能决策）

模板数量与排版功能是同质化战场，只构成及格线。本产品的组合拳是三件目前市场上没人**同时**做到的事：

1. **免费线画得比竞品狠**：渲染导出全在用户设备上、边际成本为零 → 承诺**导出永远免费、无水印**（竞品最大怨气点），以此获客；
2. **隐私即卖点**：简历不上传服务器（local-first），对求职者是真实的信任差异；
3. **AI 做成"简历教练"而非功能点**：竞品的 AI 停留在选中文本点润色；往 agent 方向走一层才是能收钱的东西 —— 工具卖不上价，服务可以。

一句话：**用"最无门槛的免费编辑器"获客，用"AI 简历教练"变现。**

## 产品原则

1. **Local-first**：编辑、渲染、导出全部发生在用户设备上。隐私是卖点，边际成本趋近于零是商业模型的地基。
2. **薄后端**：服务端只承担支付校验、AI 代理、云同步、分享链接四件事。其余一切刻意留在前端。
3. **先验证再建设**：在没有真实用户之前不搭后端；用静态托管上线，用数据决定投入。
4. **内容质量优先**：示例、文案、AI 输出按"真实优秀简历"标准维护；AI 绝不编造事实（缺数据用占位提示）。

## P0 编辑器结构性补强（当前最高优先级，全部纯前端）

这一层决定编辑器的天花板，也是后续所有功能的地基。

| 事项 | 说明 | 备注 |
| --- | --- | --- |
| 数据 schema 版本化 | 存储结构加版本号 + 迁移函数 | **隐雷**：现在改数据结构会破坏老用户数据；是未来云同步协议的雏形 |
| JSON 导入 / 导出 | 简历数据一键备份与恢复 | 缓解 localStorage 丢数据问题的第一道防线 |
| 多份简历管理（本地版） | 新建 / 复制 / 重命名 / 切换 | 一人多版本投不同岗位是核心使用模式；也是 JD 定制 agent 的落点 |
| 撤销 / 重做 | Ctrl+Z / Ctrl+Shift+Z | 目前误删条目不可恢复 |
| 自定义板块 | 板块类型库（证书 / 语言 / 获奖 / 志愿 / 自定义标题）+ 重命名 | 当前最大的单点功能差距 |
| 轻量富文本 | bullet 内加粗 / 斜体 / 链接三件套 | 「转化率提升 **27%**」的加粗是简历写作基本手法 |
| 拖拽排序 | 板块与条目支持拖拽 | 替代现有上下箭头 |

## P2 AI 能力阶梯（竞争力核心）

按深度分三级推进。L1 是工具，L3 是服务；付费上限随层级抬升。

**L1 单点生成**（✅ 润色、翻译已完成）

| 事项 | 说明 |
| --- | --- |
| 一句话生成 bullet | 「我在美团做过外卖运营」→ 3 条专业要点草稿 |
| 自动个人简介 | 读整份简历，提炼 3-5 句 summary |

**L2 分析诊断**

| 事项 | 说明 | 依赖 |
| --- | --- | --- |
| 简历评分 / 检查器 | 完整度、量化率、动词开头率、篇幅诊断 | 规则版**零后端**，先做；AI 增强版跟进 |
| JD 匹配分析 | 贴入职位描述 → 匹配度、缺失关键词、改写建议 | 现有 AI 代理即可 |
| ATS 友好度检查 | 海外求职场景的关键词与格式检查 | 随 Letter 纸张一起服务海外线 |

**L3 Agent 能力（品类分水岭，护城河所在）**

Agent = 模型 + 工具调用循环；本产品的"工具"就是简历的结构化操作（读板块、写字段、调顺序）。DeepSeek 支持 function calling，现有代理架构可直接承载。

| 事项 | 说明 | 前置 |
| --- | --- | --- |
| JD 定制 agent | 贴 JD → 分析要求 → 逐条对照 → **直接产出该岗位定制版简历**（调顺序、改措辞、突出匹配项） | 多简历管理（定制版作为新副本落地） |
| 简历教练 agent | 多轮访谈式（"这段经历最有成就感的是什么？有数据吗？"），把口语回答实时转成 bullet，逐板块把简历从 0 建出来 | 无硬前置，工程量最大 |
| 模拟面试官 | 基于简历 + JD 生成面试问题并追问 | 延伸产品，靠后 |

**落地顺序**：JD 匹配分析（单次调用跑通八成效果）→ 简历评分 → JD 定制 agent → 教练 agent。

| 事项 | 说明 |

## P4 商业化与后端（按"薄后端"原则分三步）

1. **静态上线**（0 后端）：Vercel / GitHub Pages 部署，域名，SEO 内容页（模板 / 范文落地页），第三方轻量埋点看漏斗（进入 → 编辑 → 导出）。
2. **薄后端**：AI 代理生产化（开发版 Vite 代理已跑通，生产化 = 同路由 serverless 函数 + 用户鉴权 + 计量限流 + 免费额度）+ 支付校验；用 BaaS（Supabase / 微信云开发）实现；账号体系作为支付附属品进入。
3. **同步与分享**（付费用户证明留存后）：云端多设备同步、在线简历分享链接（含访问统计）、PDF/LinkedIn 解析导入。

**免费线设计原则**：编辑、全部模板、PDF 导出永远免费无水印（获客武器）；AI 功能按额度计量收费（免费送 N 次，会员畅用）—— AI 是唯一有真实边际成本的能力，也是唯一的计费点。

## 横切事项（与阶段并行）

**工程质量**

- 测试：优先覆盖 schema 迁移、`visibleSections`、AI 响应合并（`mergeTranslated`）、导出关键路径
- React 错误边界（单个模板渲染出错不能白屏整个应用）
- 错误监控（Sentry 类）与 CI（构建 + 测试门禁）

**安全与合规（商用硬门槛）**

- API key 管理：key 只存 `.env`（已 gitignore）与生产环境变量，绝不进前端产物；泄露即轮换
- 隐私政策与用户协议；简历属敏感个人信息
- 国内上线：域名 + ICP 备案；带账号后有相应安全评估要求
- 示例内容中的真实公司名商用前替换或规避；字体商用授权核对

## 优先级逻辑

排序依据三个问题：

1. **是不是地基？** schema 版本化、多简历管理、自定义板块动数据结构，越晚做迁移成本越高 → P0
2. **要不要后端？** 不要后端的先做（检查器、DOCX、排版补全），要后端的攒到一起等验证 → P2/P4
3. **是不是竞争力？** 及格线功能补齐即可，资源向三个差异化支点倾斜：免费无水印导出、local-first 隐私、AI 教练 agent
